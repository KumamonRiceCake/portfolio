File repository RESTful API

Github Link: https://github.com/KumamonRiceCake/file-repository-RESTful-API

App on Heroku: https://kim-file-repository-api.herokuapp.com/

Features

This RESTful API provides a set of tools that allows users to save and manage files using Cloud Object Storage.

Files are uploaded and managed through Amazon Simple Storage Service (Amazon S3).

File data is accessible and manageable only by the authenticated owner.

Saved file data is tracked by the MongoDB database management system.

All endpoints are fully tested using Jest and SuperTest.


URL Endpoints

post '/file': Upload file (file size: max 20Mb). File and directory are required.

get '/file': Get the URL link of an uploaded file. Directory and filename are required.

delete '/file': Delete file or folder. Directory and filename are required.

post '/file/folders': Create folder in directory. Directory and folderName are required.

get '/file/folders': List folders in directory. Directory is required.

get '/file/directory': List files and folders in a directory. Directory is required.

delete '/file/directory': Empty directory with contents inside it. Directory is required.

get '/file/me': List all files of the user. Query strings can be provided to sort output data.

    - 'limit' and 'skip' manipulate the result shown in a page. (e.g. GET /file/me?limit=10&skip=10)
    - 'sortBy' given with 'createdAt:desc' let the result be in descending order. (e.g. GET /file/me?sortBy=createdAt:desc or GET /file/me?sortBy=createdAt:asc)

post '/users': Add a new user to the database. Name, email, and password are required.

post '/users/login': Request user login with email and password.

post '/users/logout': Logout user for an access token.

post '/users/logoutAll': Logout user for all access tokens.

get '/users/me': Get logged-in user data.

patch '/users/me': Patch logged-in user data with the data included in the request.

delete '/users/me': Delete logged-in user from database.
